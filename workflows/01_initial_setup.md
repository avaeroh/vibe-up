# Initial Setup Workflow

## Purpose
Turn a raw idea into a high-level, validated foundation that can be approved before story creation.

## Participants
- Domain Researcher
- Senior Systems Architect
- Senior Security Specialist
- Senior Business Analyst
- UX Researcher

## Steps
1. User proposes an idea.
2. Domain Researcher frames the domain and risks.
3. Architect sketches the high-level system and contracts.
4. Security Specialist flags security concerns and hard blockers.
5. UX Researcher notes obvious usability concerns.
6. BA converts the approved direction into epics.
7. User approves the foundation before stories are fleshed out.

## Output
- Problem framing
- High-level architecture
- Initial security findings
- Epics
