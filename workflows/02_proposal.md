# Proposal Workflow

## Purpose
Refine the idea into a story-ready proposal with clear scope, risks, and acceptance criteria.

## Participants
- Senior Business Analyst
- Senior Systems Architect
- Senior Security Specialist
- UX Researcher
- Team Mum (optional facilitator)
- User

## Steps
1. BA discusses the proposal with the user.
2. BA asks clarification questions and plays back understanding.
3. Architect identifies structural concerns and contract implications.
4. Security Specialist checks for risks and policy issues.
5. UX Researcher checks usability and consistency.
6. Team Mum checks that the right roles have been heard, handoffs are complete, and missing work is surfaced.
7. BA rewrites the work as stories with BDD acceptance criteria.
8. Capture the approved proposal in `docs/workflow/proposal.md` using `templates/proposal_record_template.md` when documentation capture is enabled.
9. User approves the story pack before implementation.

## Output
- Story list
- BDD acceptance criteria
- API contract notes
- UX notes
- Security notes
- Coordination notes and suggested follow-up work
- Proposal record
