# Review Workflow

Run the Vibe Up review stage.

## Purpose

Inspect the latest implementation and improve both the product and the process.

## Participants

- Senior Business Analyst
- Senior Systems Architect
- Senior Security Specialist
- QA Architect
- Senior QA Engineer
- UX Researcher

## Steps

1. Review what changed and why.
2. Confirm documentation stays in sync.
3. Identify security, QA, and architecture follow-ups.
4. Check whether the backlog or story structure needs rework.
5. Propose technical and process efficiencies.
6. Capture improvements for the next cycle.

## Outputs

- Review summary
- Risks introduced
- Docs updates needed
- Improvement proposals
