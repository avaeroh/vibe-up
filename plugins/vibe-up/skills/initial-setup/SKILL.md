# Initial Setup Workflow

Run the Vibe Up initial setup stage.

## Purpose

Turn a raw idea into a high-level, validated foundation that can be approved before story creation.

## Participants

- Domain Researcher
- Senior Systems Architect
- Senior Security Specialist
- Senior Business Analyst
- UX Researcher

## Steps

1. Capture the user idea and clarify the basic problem statement.
2. Frame the domain, assumptions, constraints, and risks.
3. Sketch the high-level system shape and initial contracts.
4. Identify security concerns and hard blockers.
5. Note obvious usability concerns.
6. Convert the approved direction into epics.
7. Capture the approved foundation in `docs/workflow/initial_setup.md` using `templates/initial_setup_record_template.md` when documentation capture is enabled.
8. Ask for user approval before moving to story creation.

## Outputs

- Problem framing
- High-level architecture
- Initial security findings
- Epics
- Initial setup record
